MecaRun Series

MecaRun is a professional robot chassis controller system project. By standardization of software and hardware, it enables quick prototyping of a robot chassis mobile system and shorten the robot development cycle.
Motivation

To address the issues of poor adaptability of robot chassis in different scenarios, high costs for customized chassis, and poor matching between ready-made controllers and chassis, our team has proposed the solution of using standardized controllers and standard components. This allows for better adaptability, lower costs, and better compatibility between different parts of the robot.

This scheme reduces repeated development work, and can quickly realize prototype and large-scale use, while ensuring a high degree of freedom. Appropriate motors and reducers, chassis structure types and chassis dimensions can be selected according to the users' own application scenarios. Finally the design can be quickly customized, or the existing equipment can be transformed into a robot chassis mobile system that meets the needs.

What we have done

Standardization refers to the establishment of uniform technical standards in terms of specifications, components, and other factors. The standardization of controllers in this project encompasses both hardware and software. The importance of standardization lies in its ability to abstract away differences in motor parameters and chassis design, providing users with a unified interface and helping them quickly achieve vehicle construction, commissioning, and mass production.

Specifically, out controller standardization includes:

Hardware standardization

1. Communication interface:

Support the control through serial port, CAN port and Ethernet port.

2. Motor interface:

The brush version of the system supports the direct control of small and medium power encoder brush motors. The external driver version supports the control of other types of motors through directional pulses, CAN, RS485, PWM/Voltage, covering a wide range of motors commonly used in mobile robot chassis. These motors are integrated into standard chassis operation commands.

Software standardization

1. Chassis parameter:

The parameters such as wheel diameter, reduction ratio, acceleration, direction, track width, forward and inverse kinematics model are standardized by software to realize rapid adaptation of different chassis.

2. Communication protocol:

The standard communication protocol of the controller is used to shield the differences of the chassis, and the secondary development programming API can be provided. The encapsulated standardized software interface can adapt to different controlling platforms.

Framework
The overall framework of the system can be divided into four parts. The first part is the input source, which is used to control the movement of the robot chassis. We use an optional type of input protocol as the control signal. The second part is the acquisition of attitude information, which can be obtained using an odometer and an optional IMU module. After gathering the attitude information, we solve the forward and inverse kinematics to obtain the acceleration. Finally, the motor movement is controlled using a triple closed-loop control system.
Ports

We offer a variety of input sources for control, including PS2, UART, USB, and SBUS protocols. For the output, MecaRun V3 provides ports for 4-way brush motors and 1-way RC servos. Additionally, MecaRun V3 also supports the extension of an IMU module as an option.

Finally, we provide host software for parameter setting and debugging, which greatly reduces the time and cost of development.
