MecaRun Series

MecaRun is a project of professional robot chassis control system. Through the standardization of software and hardware, it can quickly prototype a robot chassis mobile system and shorten the robot development cycle.
Motivation

In order to solve the problems of poor adaptability of robot chassis in different scenarios, the cost of directly customized chassis and poor matching between ready-made controller and chassis, our team proposed the solution: standardized controller + standard components.

This scheme reduces repeated development work, and can quickly realize prototype and large-scale use, while ensuring a high degree of freedom. Appropriate motors and reducers, chassis structure types and chassis dimensions can be selected according to the users' own application scenarios. Finally the design can be quickly customized, or the existing equipment can be transformed into a robot chassis mobile system that meets the needs.

What we have done

Standardization refers to the formulation of uniform technical standards in terms of specifications and components, etc. The controller standardization in this project includes hardware standardization and software standardization. The significance of standardization lies in the difference between shielded motor parameters and chassis design, which provides users with a unified upper layer interface and helps users quickly achieve vehicle construction, commissioning and mass production.

Specifically, out controller standardization includes:

Hardware standardization

1. Communication interface:

Support the control through serial port, CAN port and Ethernet port.

2. Motor interface:

The brush version supports the direct control of small and medium power encoder brush motors. The external driver version supports the control of other types of motors through directional pulses, CAN, RS485, PWM/Voltage, covering 90% of mobile robot chassis motors in the market. They are incorporated into standard chassis operation commands.

Software standardization

1. Chassis parameter:

The parameters such as wheel diameter, reduction ratio, acceleration, direction, track width, forward and inverse kinematics model are standardized by software to realize rapid adaptation of different chassis.

2. Communication protocol:

The standard communication protocol of the controller is used to shield the differences of the chassis, and the secondary development programming API can be provided. The encapsulated standardized software interface can adapt to different controlling platforms.

Framework
The overall framework can be divided into four parts. The first part is the input source, which is used for controlling the movement of robot chassis. We use an optional type of input protocol as the control signal. The second part is the acquisition of the attitude information. These information can be obtained by the odometer and an optional IMU module. After gathering the attitude information, we then solve the forward and inverse kinematics to obtain the acceleration. Finally, the motor movement is controlled through a triple closed-loop control system.
Ports

We provided rich types of input source for controlling, i.e., PS2, UART, USB, and SBUS protocol. As for the output, MecaRun V3 provides ports of 4-Way Brush Motor and 1-Way RC Servo. In addition, MecaRun V3 also supports extention of IMU module as an option.

Finally, we provide host software for parameter setting and debugging, which greatly reduce the cost of time.
